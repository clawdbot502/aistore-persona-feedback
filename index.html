Top 改进建议

1
增加场景化入口，降低认知门槛 — 在首页添加"我想做___"的场景入口（如"写文案"、"做 PPT"、"自动化客服"），而非直接暴露 Skill/MCP/Agent 等技术概念。真正的 App Store 是按用户需求组织的，不是按技术分类。
2
首页搜索框必须显眼 — App Store 的核心体验是搜索。目前首页没有明显的搜索入口，需要点到内页才能用 ⌘K。将搜索框放在首页 Hero 区下方，像 App Store 一样醒目。
3
Skill 描述中文化 — 站点是中文站（/zh），但 Skill 列表全部是英文描述和英文标签，对中文用户极不友好。至少提供中文标题和中文简介。
4
修复功能缺陷 — MCP 页面返回 404；热门搜索标签为空；首页 5 步流程的示意图都是模拟数据而非真实可点击的内容。这些空白会让用户觉得网站"半成品"。
5
增加亮色模式 + 字体大小调节 — 考虑到目标用户群不仅是年轻开发者，提供亮色模式选项和字体大小设置，让非技术用户也能舒适浏览。
6
增强筛选能力 — Skills 页面应支持按平台（Claude Code / Cursor / Copilot 等）、按难度（新手/进阶）、按语言（中文/英文）筛选，而非仅靠关键词搜索。
